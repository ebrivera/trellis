You are extracting parameters for a MATCHING workflow.

The user wants to assign/match entities from a SOURCE to a TARGET.

Available data in the database:
- people table: volunteers, visitors, donors, mentors, mentees (columns: name, email, phone, person_type, interests[], availability_days[], capacity, metadata)
- groups table: roles, small groups, initiatives (columns: name, group_type, requirements[], capacity, current_count, leader_id, metadata)

Your job is to extract:

1. SOURCE - Which entities to assign?
   - entity_type: "people" or "groups"
   - What person_type to filter? (volunteer, mentor, mentee, visitor, donor)
   - Any pre-filters? (e.g., "only those available on Sunday")

2. TARGET - What are we assigning them to?
   - entity_type: "people" or "groups"  
   - What group_type to filter? (role, small_group, initiative)
   - Any constraints?

3. MATCH_STRATEGY - How to match?
   - capacity_balanced: Balance load across targets, respect capacity limits
   - interest_overlap: Match based on interests/requirements similarity
   - proximity: Match based on location (zip codes)

4. MATCH_FIELDS - What fields to score on?
   - Common fields: interests, availability_days, zip (in metadata)
   - Weights should sum to 1.0

5. CONSTRAINTS
   - max_per_target: Which field defines capacity? (capacity, current_count)
   - min_score_threshold: Minimum match quality (0-1)

6. NOTIFICATIONS - Who gets notified after approval?
   - recipient_type: "source" (the people being assigned), "target_owners" (leaders), or "both"
   - channel: "sms" or "email"
   - template: Handlebars template with {{variables}}

---

IMPORTANT RULES:

- Use person_type filters, NOT file names (e.g., "person_type = 'volunteer'", not "volunteers.csv")
- Use group_type filters for targets (e.g., "group_type = 'role'")
- For filters, use SQL-like syntax: "availability_days @> ARRAY['Sun']" for array contains
- Default strategy is capacity_balanced
- Always include at least one notification

---

EXAMPLES:

Request: "Assign volunteers to Sunday roles based on interests"
Output:
{
  "template": "matching",
  "source": {
    "entity_type": "people",
    "filters": ["person_type = 'volunteer'", "availability_days @> ARRAY['Sun']"]
  },
  "target": {
    "entity_type": "groups",
    "filters": ["group_type = 'role'"]
  },
  "match_strategy": "interest_overlap",
  "match_fields": {
    "score_on": ["interests"],
    "weights": {"interests": 1.0}
  },
  "constraints": {
    "max_per_target": "capacity",
    "respect_preferences": true
  },
  "notifications": [
    {
      "recipient_type": "source",
      "channel": "sms",
      "template": "You're assigned to {{target_name}}! Contact {{leader_name}} at {{leader_phone}}."
    }
  ]
}
